# Реестр заявок

**Проект:** веб-приложение для отображения, фильтрации и управления реестром заявок.

**Стек:** HTML5, CSS3, JavaScript (ES6+), Bootstrap 5, Bootstrap Icons, модульная структура данных (`data.js`).

---

## Описание

Приложение отображает таблицу заявок с возможностью:

- Поиска по номеру, ИНН или названию организации.
- Фильтрации по:
  - Направлению (ЦСО, ССП, ИГС)
  - Типу объекта (Опасные, Обычные, Обычные + опасные)
  - Статусу (На проверке, В работе, На доработке, С замечаниями, Принято)
- Быстрой фильтрации по типу заявки:
  - Плановая проверка
  - Вступление
  - Изменения
- Сортировки по любому столбцу (возрастание / убывание).
- Настройки количества записей на странице (20, 50, 100, все).
- Пагинации с поддержкой «…» при большом количестве страниц.
- Отображения только закрытых заявок.
- Динамического отображения подсказок через Bootstrap Tooltip.
- Сброса всех фильтров и поиска кнопкой «Сбросить».
- Форматирования денежных значений и статусов с цветными бейджами.

## Используемые технологии

- Bootstrap 5 — сетка, компоненты, кнопки, тултипы

- Bootstrap Icons — иконки для кнопок и статусов

- Vanilla JS (ES6 Modules) — работа с DOM, фильтры, поиск, сортировка, пагинация

- CSS Custom Properties — цвета таблицы, статусов и ховер-эффектов

## Функциональные особенности

- Фильтрация и поиск: динамически фильтрует таблицу по выбранным значениям и строке поиска.

- Сортировка: любая колонка с возможностью менять порядок (asc / desc).

- Пагинация: адаптивная, с ограничением видимых страниц и кнопками ««» и «»».

- Статусы: цветные бейджи для наглядного отображения состояния заявки.

- Адаптивность: таблица и элементы интерфейса корректно отображаются на экранах мобильных и планшетах.

- Интерактивные кнопки: QuickFilter для быстрого выбора типа заявки.

## Особенности разработки

- Данные вынесены в data.js для удобного расширения и модульного подхода.

- Фильтры реализованы через dataset атрибуты (data-filter, data-value) для упрощенного связывания с JS.

- Использован debounce для поиска, чтобы минимизировать нагрузку на рендер таблицы при вводе текста.
